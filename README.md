# luci-app-microproxy

Легковесный, современный и высокопроизводительный клиент обхода блокировок для **OpenWrt** на базе прокси-ядра **Sing-box** и сетевого стека **nftables + TPROXY**.

Данный плагин разработан как простая, быстрая и чистая альтернатива PassWall и OpenClash, полностью оптимизированная под современный протокол **VLESS + REALITY (TCP / XHTTP)**.

---

## ⚡ Быстрая установка и удаление

Для установки плагина вам не нужно собирать его из исходников — вы можете развернуть его на вашем роутере одной командой прямо по SSH.

### 📥 Установка в один клик
Подключитесь к вашему роутеру по SSH и выполните команду:
```bash
wget -qO- https://raw.githubusercontent.com/AlexeDark/luci-app-microproxy/main/install.sh | sh
```

### 🗑️ Полное удаление
Если вы хотите полностью очистить систему от плагина и его настроек (включая автоматическое восстановление портов dnsmasq и очистку правил фаервола):
```bash
wget -qO- https://raw.githubusercontent.com/AlexeDark/luci-app-microproxy/main/uninstall.sh | sh
```

## 🚀 Основные возможности и архитектура

### 1. Архитектура Маршрутизации: Direct-by-Default
В отличие от тяжелых плагинов, которые пускают весь трафик через прокси (исключая РФ), **microproxy** работает по схеме **Direct-by-Default**. Весь интернет-трафик идет напрямую через вашего провайдера на максимальной скорости. Через прокси уходят **исключительно** домены и подсети из вашего белого списка (whitelist).
* Это предотвращает ложные срабатывания антифрод-систем (мобильные банки, Госуслуги, маркетплейсы не заблокируют ваш аккаунт, так как их скрытая CDN-аналитика не улетит через зарубежный IP прокси).

### 2. Нативный обход блокировок (nftables TPROXY)
Никакого устаревшего iptables. При старте службы создается атомарная таблица `inet nft_microproxy` с механизмом TPROXY. В прокси-ядро Sing-box перенаправляются только пакеты, идущие на:
* Выделенный диапазон Fake-IP (`198.18.0.0/15`).
* IP-адреса и подсети из статических подписок и персонального списка (`@whitelist_ips`).
* Временно добавленные IP-адреса (`@dynamic_ips`).

### 3. Автоматический Failover (Обсерватория Sing-box)
Плагин использует встроенный механизм **Observatory** ядра Sing-box для фонового замера задержек (RTT) до проверочного URL через все ваши прокси-серверы. Групповой outbound типа `urltest` автоматически направляет трафик на сервер с наименьшим пингом (**leastPing**), защищая от «дребезга» каналов с помощью порога чувствительности (`tolerance`). Никаких тяжелых Bash-скриптов проверки связи!

### 4. Умный JS-парсер ссылок VLESS Reality + XHTTP
При импорте ссылок во вкладке «Серверы» умный парсер на стороне браузера автоматически проверяет параметры. Если в ссылке используется современный быстрый транспорт **XHTTP**, но провайдер забыл настроить параметры маскировки, плагин **принудительно дописывает** обфускацию:
```json
"mode": "packet",
"x_padding_bytes": "100-1000"
```
Это зашумляет размер пакетов и предотвращает детектирование ТСПУ сигнатур Reality-XHTTP соединений.

### 5. Безопасный динамический обход (On-Demand Proxy)
Если нужный вам сайт перестал открываться, просто введите его домен на главной странице. Плагин автоматически:
1. Разрешит домен в IP-адреса с помощью `nslookup`.
2. Защитит от ложных блокировок: **отклонит** добавление, если домен принадлежит зоне `.ru`/`.рф` (во избежание проблем с банками).
3. Добавит полученные IP в динамическое nftables-множество с таймаутом жизни **12 часов**. Через 12 часов правила автоматически очистятся из оперативной памяти роутера.

### 6. Бесшовная DNS-интеграция
* **Без AdGuard Home**: При старте службы системный `dnsmasq` автоматически сдвигается на порт `54`, а Sing-box занимает основной порт `53`. Локальные имена LAN резолвятся через dnsmasq, а whitelisted-домены мгновенно получают Fake-IP. При остановке порт 53 возвращается dnsmasq.
* **С AdGuard Home**: Sing-box биндится на свободный порт `7913`. AdGuard Home работает на основном порту 53, а в качестве Upstream DNS вы указываете `127.0.0.1:7913`.
